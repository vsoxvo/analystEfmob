## Блок 1: Теория вероятности и логика

### Задание 1: Фермер
**Условие:**
На ферме содержатся шесть разных видов животных, и каждый раз, когда фермер заходит в сарай, он видит одно случайное животное. За день фермер заходит в сарай 6 раз.
Каково математическое ожидание количества разных видов животных, которые фермер увидит за день?

* **Решение:**
  
1. **Вероятность НЕ встретить** животное за один заход: $5/6$.
2. **Вероятность НЕ встретить** животное за все 6 заходов: $(5/6)^6 \approx 0.3349$.
3. **Вероятность встретить** животное хотя бы раз: $1 - 0.3349 = 0.6651$.
4. Так как видов всего 6, и условия для всех одинаковые, мы просто умножаем эту вероятность на количество видов.

**Расчет:** $E = 6 \cdot 0.6651 = 3.99$.

### Задание 2: Кулинарное соревнование
**Условие:**
В конкурсе участвуют 80 шеф-поваров с уникальными уровнями мастерства. В первом этапе судьи случайным образом распределяют их по парам (в любом состязании двух шефов выигрывает тот, у кого выше уровень мастерства). На втором этапе шефы снова случайно образуют пары для финального раунда (пары могут повториться). Победную награду получают те, кто выиграл в обоих этапах.
Каково математическое ожидание числа победителей?

* **Решение:**
  
1. Каждому шефу присвоим ранг $k$ (от $1$ до $80$), где $80$ — самый сильный.
2. Для шефа с рангом $k$ количество более слабых соперников равно $k-1$, а общее число возможных соперников — $79$.
3. Вероятность победы в одном раунде: 
   $$P_k = \frac{k-1}{79}$$
4. Поскольку раунды независимы, вероятность победы в двух этапах: 
   $$P(\text{win}_k) = \left(\frac{k-1}{79}\right)^2$$
5. Математическое ожидание ($E$) суммы индикаторов (победителей) равно сумме их вероятностей:
   $$E = \sum_{k=1}^{80} \left(\frac{k-1}{79}\right)^2 = \frac{1}{79^2} \sum_{m=0}^{79} m^2$$
6. Используя формулу суммы квадратов $\sum m^2 = \frac{n(n+1)(2n+1)}{6}$ для $n=79$:
   $$E = \frac{79 \cdot 80 \cdot 159}{6 \cdot 79^2} = \frac{167\,480}{6241} \approx 26.83$$

**Ответ:** 26.83.


### Задание 3: Одинокая дорога

**Условие:**
На пустынном шоссе вероятность появления автомобиля за 30-минутный период составляет 0.95.
Какова вероятность его появления за 10 минут? А за 27 минут?

* **Решение:**
1. **Тишина за 30 минут:** Если машина приезжает с вероятностью $0.95$, значит, вероятность полной тишины (ноль машин) — это $1 - 0.95 = 0.05$.
2. **Тишина за 1 минуту:** Чтобы найти вероятность тишины за меньший срок, нужно извлечь корень: $0.05^{1/30}$.
3. **Расчет для 10 минут:** - Вероятность тишины: $0.05^{10/30} = 0.05^{1/3} \approx 0.368$.
   - Вероятность появления машины: $1 - 0.368 = 0.632$ ($63.2\%$).
4. **Расчет для 27 минут:** - Вероятность тишины: $0.05^{27/30} \approx 0.067$.
   - Вероятность появления машины: $1 - 0.067 = 0.933$ ($93.3\%$).
  
**Ответ:** 63.2; 93.3

### Задание 1: Изоморфизмы
**Условие:**
Реализовать функцию (или тело функции), которая проверяет на изоморфность два слова. Пояснение: строки s и t называются изоморфными, если все вхождения каждого символа строки s можно последовательно заменить другим символом и получить строку t. Порядок символов при этом должен сохраняться, а замена — быть уникальной. Так, два разных символа строки s нельзя заменить одним и тем же символом из строки t, а вот одинаковые символы в строке s должны заменяться одним и тем же символом.


**Пример:**

```python
s = 'paper' 
t = 'title' 
print(is_isomorphic(s, t))
# Вывод: 
True
```
Оценить оптимальность решения по времени и памяти и прикрепить текст кода.


**Решение:**

```python
def is_isomorphic(s: str, t: str) -> bool:
if len(s) != len(t):
return False
mapping = {}
used = set()
for cs, ct in zip(s, t):
if cs in mapping:
if mapping[cs] != ct:
return False
else:
if ct in used:
return False
mapping[cs] = ct
used.add(ct)
return True

# Время: O(n).
# Память: O(min(n, alphabet_size)).
```

### Задание 2: Натуральная последовательность
**Условие:**
Реализовать функцию (или тело функции), которая находит единственное отсутствующее число из последовательности натуральных чисел 1,2,…,n.


**Пример:**
```python
nums = [1, 2, 3, 4, 5, 6, 8, 9, 10, 11]
print(missing_number(nums))
# Вывод: 
7
```
Оценить оптимальность решения по времени и памяти и прикрепить текст кода.


**Решение:**
```python
def missing_number(nums: list[int]) -> int:
# n = len(nums) + 1
n = len(nums) + 1
expected = n * (n + 1) // 2
return expected - sum(nums)

# Оценка:
# Time: O(n) — проход по списку для суммы и поиска max.
# Memory: O(1) — используем только переменные для хранения сумм.
```
### Задание 3: Факторизация
**Условие:**
Реализовать функцию (или тело функции), которая при введении натурального числа n разбивает его на простые множители (представить его в виде простых чисел).

**Пример:**

```python
n = 56
print(prime_factors(n))
# Вывод:
[2, 2, 2, 7]
```

**Решение:**

```python
def prime_factors(n: int) -> list[int]:
factors = []
# делим на 2
while n % 2 == 0:
factors.append(2)
n //= 2
# пробуем нечетные делители до sqrt(n)
p = 3
import math
limit = math.isqrt(n) + 1
while p <= limit and n > 1:
while n % p == 0:
factors.append(p)
n //= p
limit = math.isqrt(n) + 1
p += 2
if n > 1:
factors.append(n)
return factors

# Оценка:
# Time: O(sqrt(n)) — цикл работает до корня из n.
# Memory: O(log n) — количество простых множителей.

```

### Задание 3: Покупки

**Условие:**

```sql

create table account
(
    id integer, -- ID счета
    client_id integer, -- ID клиента
    open_dt date, -- дата открытия счета
    close_dt date -- дата закрытия счета
)

create table transaction
(
    id integer,  -- ID транзакции
    account_id integer,  -- ID счета
    transaction_date date,  -- дата транзакции
    amount numeric(10,2), -- сумма транзакции
    type varchar(3) -- тип транзакции
)
```

Вывести ID клиентов, которые за последний месяц по всем своим счетам совершили покупок меньше, чем на 5000 рублей.
Без использования подзапросов и оконных функций.

**Запрос:**
```sql
SELECT a.client_id
FROM account a
JOIN transaction t ON a.id = t.account_id
WHERE t.transaction_date >= CURRENT_DATE - INTERVAL '1 month'
GROUP BY a.client_id
HAVING SUM(t.amount) < 5000;
```
